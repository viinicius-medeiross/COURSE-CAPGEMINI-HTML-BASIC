Um servidor é um computador no qual o seu web host armazena os arquivos e os bancos de dados necessários para o seu site. Quando um visitante quer acessar o seu site, o navegador deles envia uma solicitação para o seu servidor, e então transfere os dados necessários através da internet.

A hospedagem VPS oferece a você um servidor virtual que simula um servidor físico. Contudo, na realidade, a máquina está sendo compartilhada entre diversos usuários.

Usando uma tecnologia de virtualização, o seu provedor de hospedagem instala uma camada virtual sobre o sistema operacional (SO) do servidor. Essa camada divide o servidor em partições e permite que cada usuário instale o seu próprio sistema operacional e software.

Portanto, um servidor virtual privado (VPS) é tanto virtual quanto privado, já que você tem controle completo. Ele é separado dos servidores de outros usuários num nível de sistema operacional.

De fato, a tecnologia de VPS é similar a criar partições no seu próprio computador, quando você quer rodar mais de um sistema operacional (por exemplo Windows e Linux) sem reiniciar o PC.

Rodar uma VPS permite que você configure o seu site dentro de um container seguro com recursos garantidos (memória, espaço em disco, núcleos de CPU, etc.), que você não precisa compartilhar com outros usuários.


Com a hospedagem VPS, você tem o mesmo acesso de nível root que teria num servidor dedicado, mas com um preço muito menor.

--------------- RESUMO GERAL DOS ESTUDOS --------------


Estrutura básica : Um documento HTML começa com uma declaração do tipo de documento (doctype) seguida pela tag <html>, que contém duas seções principais: <head>e <body>. A seção <head>é usada para incluir metadados, como o título da página, links para arquivos CSS ou JavaScript e outras informações importantes. A seção <body>contém o conteúdo visível da página, como texto, imagens, links e elementos interativos.

Tags : As tags HTML são usadas para marcar diferentes partes do conteúdo da página e definir sua estrutura. Cada tag é escrita entre colchetes angulares < >. Algumas tags não precisam de uma tag de fechamento e são chamadas de tags vazias, por exemplo, uma tag <img>para imagens.

Atributos : As tags HTML também podem ter atributos que fornecem informações adicionais sobre o elemento. Os atributos são adicionados à tag de abertura e fornecem valores entre aspas, por exemplo, o atributo hrefusado em um link.

Textos : Para adicionar texto ao documento, você pode usar uma tag <p>para parágrafos, <h1>a <h6>para cabeçalhos com diferentes níveis de importância, <strong>para texto em negrito, <em>para texto em itálico, etc.

Links : Para criar links para outras páginas da web ou recursos, você pode usar uma tag <a>. O atributo hrefé usado para especificar a URL de destino.

Imagens : Para exibir imagens na página, você pode usar uma tag <img>. O atributo srcé usado para especificar o caminho da imagem.

Listas : Existem dois tipos principais de listas em HTML: listas ordenadas <ol>e listas não ordenadas <ul>. Cada item da lista é marcado com uma etiqueta <li>.

Formulários : Os formulários em HTML permitem que os usuários enviem dados para um servidor web. Os elementos de formulário incluem <input>, <textarea>, <select>, entre outros.

Comentários : Você pode adicionar comentários no código HTML usando <!-- Comentário aqui -->. Os comentários não são exibidos na página, mas são úteis para fornecer informações sobre o código para outros desenvolvedores ou para referência futura.

Semântica : HTML5 suporta novas tags semânticas, como <header>, <nav>, <main>, <footer>, etc., que ajudam a fornecer um significado mais claro ao conteúdo, além de melhorar a acessibilidade e a indexação dos mecanismos de busca.
